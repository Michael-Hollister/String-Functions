SF RegEx_Matches

Evaluates the source string and fills the internal list of successful matches.

  Syntax 
Return Count As Integer = SF RegEx_Matches( Str As String, Pattern As String )
  Parameters 
Str
Source string. 
Pattern
The regular expression.

  Returns

Returns the number of successful matches that matched the expression.
  Exceptions

Code Exception Condition
208 InvalidExpression
 Pattern is a invalid expression.
210 UnknownError
 Occurs if the complexity of matching the expression against an N character string begins to exceed O(N2), or if the program runs out of stack space while matching the expression, or if the matcher exhausts its permitted memory allocation. (this exception is unlikely to happen)
  Description

Evaluates the source string and fills the internal list of successful matches and returns the match count. The information for each match can be retrieved by the commands, SF RegEx_GetMatchCount, RegEx_GetMatchPosition and RegEx_GetMatchStr.

  Example Code 
Sentence$ = "In computing, a regular expression provides a concise and flexible means to " + Chr$(34) + "match" + Chr$(34) + " (specify and recognize) strings of text"
Pattern$ = "[A-Za-z]+" // Find one or more characters that are A-Z or a-z.

Print "Here are the following words contained in this sentence:"
Print Sentence$
Print

Rem Get the number of matches in the sentence.
MatchCount = SF RegEx_Match(Sentence$,Pattern$)

Print "The sentance contains " + Str$(SF RegEx_GetMatchCount()) + " words."
Print

Rem Iterate through the words listing them on the screen.
For X = 0 To MatchCount-1
    Print "Word: " + SF RegEx_GetMatchStr(X) + " | Position: " + Str$(SF RegEx_GetMatchPosition(X))
Next X

Wait Key
